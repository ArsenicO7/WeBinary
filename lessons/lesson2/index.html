Lesson 2: What is Binary?

Binary is a base 2 number system consisting of only two types of digits: 0 and 1. While it is not used in everyday math calculations, binary is the foundation of computer science, as all computers operate on a fundamental level using this number system.


Computers and Binary


Computers store data, on the most basic level, in the form of bits. A single bit is an amount of space in a computer equal to one binary digit, meaning it can be set to a value of 0 or 1. A computer operates using collections of thousands of thousands of bits, each with a different value of 0 or 1. These combinations create different commands or forms of information for the computer to interpret, allowing it to perform a multitude of tasks and store vast amounts of information.


For example, let’s say we have a two-bit piece of data. This means there are two different binary digits, and using the formula in the previous lesson we have 2^2 = 4 possible combinations with these two bits: 00, 01, 10, and 11. The computer can use these four different combinations to record different tasks or interpret functionality. For instance, these particular two bits can represent the state of a document. If the left bit is 1, then the user is currently editing the document. If the right bit is 1, the document has unsaved changes. Otherwise, the bits have a value of 0. Therefore, if this two-bit piece of data is 00, the file is closed with no changes made since the last save. If the value is 11, then the document is in the middle of being edited.


Of course, the combinations are rarely as simple as this. Rather, a task like this would most likely have activity and file states recorded with bits upon bits rather than a mere two bits. However, computer scientists do not need to know every single one of these binary communications or even account for them when creating software and altering program functionality, since binary is the very backend of what the modern programmer deals with. Rather, the programmer deals with a higher end language, such as C++, that alters the lower level languages like Assembly all the way down to the computer’s binary computation.


Counting With Binary


Binary, just like all other number systems, can also be used to count numbers. All number systems count depending on their base value. Decimal, with a base of 10, counts numbers at intervals of 10. The rightmost digit will increment a numbers value by 10^0 = 1. The next digit will increment number values by 10^1 = 10. When we have the number 11, increasing the right digit by one to 2 will make the number 12, which is an increase by a single value of 1. If the digit to the left of it is increase instead to create 21, the value of the number is increased by 10.


Binary works the same way, but instead at increments of 2, its base value. With the number 11 in binary, the right most digit actually has a value of 2^0 = 1. The digit to the left of it will have a value of 2^1 = 2. Therefore, increasing the binary value of 00 to 11 increases it by one with the right digit by 1 and the left digit by 2. Thus, the total value of 11 in binary is 1 + 2 = 3.


The same concept applies to larger binary numbers. Let’s say we have the number 101011. We know that the rightmost digits carry the values of 1 and 2 for a total value of three. The next digit, which is a 0, can hold a value of 2^2, or 4. Going to the next digit, the number’s value at 2^3 = 8 is 1, meaning the total value of the number 1011 is 8 + 2 +1 = 11. Going to the left, the value of the 2^4 = 16 bit is 0, so we do not add any value at this digit. For the leftmost value, the position is 2^5 = 32, so the 1 digit at this location creates the addition of 32 to the number’s value. Therefore, the decimal value of the binary number 101011 is (2^5)*1 + (2^4)*0 + (2^3)*1 + (2^2)*0 + (2^1)*1 + (2^0)*1 = 32 + 8 + 2 + 1 = 43.


The math can be even more straightforward by using the doubling trick for counting binary. The rightmost value has a value of 1, the value left of it 2, then 4, 8, 16, and so on. Each of these values is a double of the previous, so rather than needing to work with exponential equations for calculating a binary number in decimal, we can simply count by twos for each number that has a 1.


To demonstrate, say we have the binary number 100111. The right digit has a 1 at the binary location of 1, then the next digit is 1 at the binary location of 2. After that is 4, so we so far with the first 3 rightmost digits have 1, 2, and 4. The next value is 8, but there’s a 0 digit there, so that value is skipped when counting. The same goes for the next digit at 16. Finally, we have a 1 at the value 32 location. Thus, the final value of the number 100111 in decimal is 1 + 2 + 4 + 32 = 39.


What about going from decimal to binary? Since all binary digits are doubles of the previous number and are restricted to containing certain values, this can easily be done using a strategy of continuously subtracting the largest possible value from the number. For example, to get the binary value of 39, the largest binary digit value that can fit inside it is 32, which is 2^5. The binary value for 32 is 100000, and that leaves only 39 - 32 = 7 digits left to add to the binary number. We then have to work our way right of the first digit in the number. The next value after the 32 value digit is 16. 16 is larger than 7, so that digit cannot be used and is left as a 0. Next is 8, which also cannot be used. However, the next digit is 4, which can fit within 7, so that digit is turned into a 1, leaving the binary number as 100100 with 7 - 4 = 3 left to add to the binary number. The next digit value is 2 and the final digit value is 1, adding up to 3, so the last step is setting the two rightmost digits both to 1. Therefore, 39 in binary is 100111.


This can be useful when dealing when binary is used to count in computers. After all, computers need to count the length of tasks or the number of times a task has been performed, which will be done in binary, not decimal. Being told that the time that passed since a task was performed as 101110111 does not really have any meaning to us, but being told that 375 seconds have passed does. We can then use this knowledge to adjust parameters in a program, determine when to reset the counter, or whatever else it is the programmer needs to do.
