Lesson 3: Adding and Subtracting in Binary

All number systems can have the same mathematical operations performed on them, and binary is no exception. There are two methods we can use to add and subtract with binary: converting to decimal and calculating or manipulating the binary digits themselves.


Using the information in the previous lessons to count binary digits, or convert binary to decimal, we can simply convert the binary digits that need to have the operation performed and then perform the addition or subtraction in decimal. For example, let’s say we have to add the two numbers 10111 and 1100. We can figure this out easily enough by converting each of them to decimal and then performing the addition. 10111 in decimal is 1 + 2 + 4 + 16 = 23 and 1100 in decimal is 4 + 8 = 12. Hence, the answer is 23 + 12 = 35. Converting back to binary with 35, the highest number is 32. After that there is only 3 left, which are the two rightmost binary digits set to 1. Thus, 10111 + 1100 = 100011 in binary.


However, binary digits can also be added as they are using the same methods we use to add multiple digits in decimal. If we line the digits up, we can perform math the same way, simply using the values of another base system.


This method can be useful if you want to avoid conversion, although it all depends on one’s preference to decide which method to use: convert and then operate, or operate directly in the binary number system. Both methods are acceptable to perform binary addition and subtraction.


It should be noted that often when one is expected to operate on numbers in a number system other than decimal that the answer desired may also be in the different number system. As in, if an operation is performed on binary numbers, the answer may need to be in binary, although it could also need to be in decimal or another number system. One should always be careful to look at the number system specified for the numbers that are being operated on and the number system the solution is expected to be in.
