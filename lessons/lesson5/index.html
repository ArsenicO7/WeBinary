Lesson 5: What is Hexadecimal?

Hexadecimal is the name of the base 16 number system. This means that each digit in this number system can hold a total of sixteen values. For decimal, which can hold up to ten values in each digit, the numbers range from 0-9, but in hexadecimal how are numbers higher than 9 expressed in a digit? This is actually done using letters. The value A in a digit is equal to the number 10 in decimal, B is equal to 11, and so on to F is equal to 15. Each hexadecimal digit can range from 0-F, for a total of 16 possible digits.


Since hexadecimal is just another number system, the same methods can be used for converting between different number systems, such as binary and decimal. Converting from hexadecimal to decimal is very similar to how we convert from binary to decimal. The main added difficulty is simply the larger size of the digits.


The rightmost digit of any hexadecimal digit can have a value of 0-F, but then what does that value become for the digits left of it? The base of the number system is 16, so the next value will hold values of 16^1, or increment the number by 16. So for the number 10 in hexadecimal, we have the 16 value digit with a value of 1, so the total number of the hexadecimal number 10 is 16 in decimal. If the digit increases and we have the number 20, another 16 value is added for a total value of 32 in terms of decimal. For the hexadecimal number F0, we have a total number value in decimal of 16 * 15 = 240. The next digit will increment the number values by 16 ^ 2 = 256. That means the number 100 in decimal in hexadecimal is 256 in decimal.


For an example with a more complex hexadecimal number, let’s say we have the number D04A. The first digit on the left will be the 16^3 = 4096 digit, and the digit D is the 13th digit that this digit can hold. That means the total value of the digit for this number, or the value of D000, is 4096 * 15 = 61,440, which is quite the large number compared to what we would find with only four digits in binary or decimal. The next digit is a 0, so there is nothing for the 256 digit. Next there is a 4 at the 16 value digit, adding a value of 16 * 4 = 64 to the total number, which is now 61,440 + 64 = 61,504. The final digit is A, which represents a value of 10 in decimal, so the final value of D04A in decimal is 61,504 + 10 = 61,514.


Going from decimal to hexadecimal can be a bit trickier do to having to find the highest number that can go for a leftmost digit. In binary, one can simply double by twos to find the maximum digit, but now everything is in terms of multiples of 16. However, the same steps can still be used, it just requires knowing all of the multiples of 16 for as large as the number requires, which is not easy. As a simpler solution, the next chapter discusses how hexadecimal can be converted to binary and then converted to decimal, which is actually much easier than converting straight from hexadecimal to decimal.
